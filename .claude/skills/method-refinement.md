# Purpose

Improve the technical framing and novelty of EvoGuard.

# Workflow

1. Read `docs/method_design.md` and `CLAUDE.md`.
2. Identify unclear assumptions.
3. Check whether the novelty over ToolSafe, MOSAIC, EAPO, MAGIC, RUBAS, and AgentWatcher is explicit.
4. Improve:
   - Problem formulation.
   - Threat model.
   - Algorithm description.
   - Reward design.
   - Training loop.
   - Evaluation setup.
5. Produce:
   - Revised method section.
   - List of remaining weaknesses.
   - Suggested ablations.
